# 기도의샘물 전체 리모델링 계획

## 1. 목표

`기도의샘물`을 단순 정적 기도문 모음이 아니라, 유튜브 채널과 연결된 기독교 묵상 콘텐츠 허브이자 상황별 기도문 PDF 판매 랜딩으로 리모델링한다.

핵심 목표는 다음과 같다.

- 매일 바뀌는 기도문과 말씀 묵상으로 방문자가 위로를 얻는다.
- 유튜브 영상, 기도문, PDF 상품, 기도제목 나누기가 자연스럽게 이어진다.
- 40~70대 독자도 편하게 읽을 수 있는 큰 글씨, 높은 대비, 명확한 버튼 중심 UI를 만든다.
- 첫 수익화는 상황별 기도문 PDF 패키지 판매 랜딩으로 시작한다.
- 향후 React, 결제 연동, 오디오팩, 30일 챌린지, 맞춤 기도문 생성까지 확장 가능한 데이터 구조를 유지한다.

## 2. 현재 상태

- 사이트는 정적 HTML/CSS/JavaScript 기반이다.
- 주요 콘텐츠 데이터는 `data/*.js`에 분리되어 있다.
- 실제 로드되는 공통 스타일과 스크립트는 `assets/css/style.css`, `assets/js/script.js`다.
- `generate_site.py`는 한글 인코딩이 깨져 있을 수 있으므로 실행하지 않는다.
- `AGENTS.md`가 프로젝트 운영 규칙으로 추가되어 있다.
- PDF 판매 계획 구현을 위해 `data/pdfProducts.js`와 `assets/js/script.js` 일부 렌더링 코드가 추가된 상태다.

## 3. 리모델링 원칙

- 브랜드 표기는 `기도의샘물`로 통일한다.
- 모든 핵심 페이지는 기도문, 유튜브 영상, PDF 상품, 기도제목 나누기 중 최소 하나의 다음 행동으로 이어져야 한다.
- 외부 이미지는 무단 사용하지 않는다. Pinterest는 디자인 참고만 하고 실제 삽입 이미지는 직접 제작, 생성 이미지, 라이선스 확인 이미지, 자체 아이콘으로 대체한다.
- 사랑의교회 사이트처럼 정돈된 정보 구조, 절제된 색상, 신뢰감 있는 종교 콘텐츠 톤을 참고하되 복제하지 않는다.
- 기능은 현재 정적 구조에서 구현하되, 데이터와 UI를 분리해 React 전환이 쉬운 형태로 만든다.
- 실제 결제 연동 전에는 외부 결제 링크 또는 문의 페이지를 사용한다.

## 4. 정보 구조

최종 사이트의 핵심 메뉴는 다음 순서로 정리한다.

1. 홈
2. 오늘의 기도문
3. 밤기도
4. 말씀 묵상
5. 유튜브 영상
6. PDF 패키지
7. 기도제목 나누기
8. 소개/문의

기존 페이지는 다음 역할로 정리한다.

- `index.html`: 전체 허브, 오늘의 기도, 유튜브, PDF 상품 진입점
- `prayers.html`: 상황별 기도문 검색과 카테고리 카드
- `night-prayer.html`: 수면/밤기도 특화 랜딩
- `morning-prayer.html`: 아침기도 특화 페이지
- `meditation.html`: 말씀 묵상 아카이브
- `videos.html`: 유튜브 영상 아카이브와 관련 PDF 연결
- `premium-pdf.html`: 1차 수익화 핵심 페이지
- `prayer-request.html`: 기도제목 나누기
- `prayer-cards.html`: 말씀/기도 카드 자료
- `prayer-challenge.html`: 30일 챌린지 예고 및 확장 페이지

## 5. 1차 MVP 범위

1차 리모델링은 실제 결제 시스템이나 PDF 파일 제작까지 가지 않고, 판매 가능성을 검증하는 랜딩 중심으로 만든다.

포함 범위:

- 홈 첫 화면 리디자인
- 상황별 기도문 카드 UI 개선
- PDF 패키지 판매 랜딩 개편
- 유튜브 영상과 PDF 패키지 연결 CTA 추가
- 기도제목 나누기 신뢰감 개선
- 모바일 가독성 개선
- SEO 메타와 내부 링크 정리

제외 범위:

- 사이트 내 PG 결제 연동
- 실제 PDF 파일 다운로드 자동 제공
- 회원가입/로그인
- AI 맞춤 기도문 생성
- 오디오 파일 판매 자동화
- React 전환

## 6. 수익화 로드맵

### 1순위: 상황별 기도문 PDF 패키지

대표 상품:

- 잠들기 전 30일 밤기도 PDF
- 마음 위로 기도문 모음
- 불안 내려놓기 기도문
- 가족을 위한 기도문
- 자녀를 위한 30일 기도문
- 큰 글씨 기도문 PDF

상품 구성:

- 기도문
- 말씀구절
- 짧은 묵상
- 적용 질문
- 오늘 붙들 한 문장
- 인쇄용 안내
- 휴대폰 저장용 안내

가격 운영 기준:

- 가격이 확정되기 전에는 가격과 구매 버튼을 노출하지 않는다.

- 판매 가능한 자료는 상품별 가격, 주문 기록, 구매 완료 권한을 기준으로 제공한다.

결제 흐름:

- 1차: 공개 미리보기와 자료 문의
- 2차: 자료별 단건 주문 생성과 결제 승인
- 3차: 결제 완료 뒤 즉시 다운로드 및 내 자료실 재이용

## 7. 데이터 구조 계획

현재 데이터 파일을 유지하되 다음 파일을 추가/정리한다.

- `data/prayers.js`: 무료 기도문
- `data/meditations.js`: 말씀 묵상
- `data/videos.js`: 유튜브 영상
- `data/prayerCards.js`: 카드 자료
- `data/prayerChallenge.js`: 챌린지
- `data/pdfProducts.js`: PDF 상품
- `data/dailyContent.js`: 오늘의 기도/말씀/영상 매칭

`pdfProducts.js` 기본 구조:

```js
{
  id: "night-prayer-30",
  category: "밤기도",
  title: "잠들기 전 30일 밤기도 PDF",
  audience: "잠들기 전 마음이 복잡한 분",
  composition: ["기도문 30편", "말씀구절", "묵상질문"],
  resourceId: null,
  salesStatus: "inquiry",
  priceKrw: null,
  sampleUrl: "prayer-cards.html",
  isFeatured: true
}
```

`dailyContent.js` 기본 구조:

```js
{
  dateKey: "auto",
  prayerId: "p01",
  meditationTitle: "요한복음 14장 27절 묵상",
  videoTitle: "염려가 내려놓아지지 않는 밤",
  productId: "night-prayer-30"
}
```

1차에서는 날짜 기반 자동 계산으로 매일 다른 콘텐츠를 보여준다. 서버가 없으므로 `new Date()`와 배열 인덱스 순환 방식으로 구현한다.

## 9-1. 이미지 없이 완성하는 시각 디자인 시스템

### 목표

현재 단계에서는 실제 사진 이미지 파일을 사용하지 않는다. 그럼에도 사이트가 비어 보이거나 임시 페이지처럼 보이면 안 된다. `기도의샘물`은 CSS 그라데이션, radial light, pseudo-element, inline SVG, 카드, 배너, 도형 시스템만으로도 고급스러운 기독교 말씀·기도 콘텐츠 미디어처럼 보여야 한다.

시각 목표는 다음과 같다.

- 단순 텍스트 중심 페이지를 벗어나 사진 없이도 완성도 높은 콘텐츠 사이트처럼 보이게 한다.
- 조용함, 따뜻함, 평안함, 신뢰감, 프리미엄한 인상을 만든다.
- 대형 교회 홈페이지의 공식적인 신뢰감은 참고하되, 교회 행정 사이트처럼 딱딱하지 않게 한다.
- 잠잘 때 듣는 기도, 말씀 묵상, 아침기도에 어울리는 부드러운 빛, 물결, 종이 질감, 새벽빛, 촛불 느낌을 추상적으로 표현한다.
- 나중에 직접 제작 이미지나 AI 생성 이미지를 넣더라도 현재 구조를 크게 바꾸지 않고 교체할 수 있게 한다.

### 저작권 원칙

- `sarang.org`, Pinterest, 무료 이미지 사이트, 타 사이트의 이미지, 로고, 아이콘을 복사하지 않는다.
- 외부 이미지 URL을 임의로 삽입하지 않는다.
- 무료 이미지 사이트 이미지를 사용하더라도 라이선스 확인 전에는 넣지 않는다.
- 지금 단계에서는 CSS/SVG 기반 자체 디자인만 사용한다.
- 향후 이미지 삽입 위치는 HTML 주석으로만 남긴다.
- 사용자 화면에는 `준비 중`, `샘플`, `임시 이미지` 같은 표현을 노출하지 않는다.

주석 예시:

```html
<!-- 향후 히어로 배경 이미지 삽입 위치 -->
<!-- 향후 말씀 카드 실제 이미지 삽입 위치 -->
<!-- 향후 30일 기도 챌린지 대표 이미지 삽입 위치 -->
```

### 히어로 배너 시스템

히어로는 실제 사진 대신 레이어드 CSS 배경으로 만든다.

- 딥 네이비, 아이보리, 베이지, 은은한 골드가 자연스럽게 섞인 gradient를 사용한다.
- `::before`, `::after`로 부드러운 빛 번짐, 원형 glow, 물결 라인, 종이 질감 느낌을 만든다.
- 밤기도/수면기도 계열은 새벽빛, 촛불, 잔잔한 물결을 추상적으로 표현한다.
- 십자가 실루엣을 넣을 경우 아주 절제된 pseudo-element로 처리하고, 본문보다 강하게 보이지 않게 한다.
- 텍스트 대비가 낮아지지 않도록 히어로 본문 영역에는 충분한 명도 차이를 둔다.
- 모바일에서는 decorative layer가 텍스트를 가리지 않게 opacity와 위치를 조정한다.

구현 기준:

- 공통 클래스 후보: `visual-hero`, `hero-glow`, `hero-wave`, `hero-cross-mark`
- 실제 이미지 교체 여지를 위해 히어로 내부에 향후 이미지 삽입 주석을 남긴다.
- 배경 장식은 CSS 장식이므로 접근성 트리에 노출하지 않는다.

### 아이콘 시스템

외부 이미지 파일 대신 inline SVG 또는 CSS icon class를 사용한다. 아이콘은 얇고 우아한 라인 스타일로 통일한다.

필수 아이콘:

- 기도
- 말씀
- 수면기도
- 아침기도
- 영상묵상
- 말씀카드
- 30일 챌린지
- 프리미엄 PDF
- 기도제목
- 문의하기

아이콘 규칙:

- 색상은 골드, 네이비, 세이지 그린 중심으로 제한한다.
- stroke width, icon size, badge size는 CSS 변수로 관리한다.
- 콘텐츠 카드 우측 상단에 작은 SVG 아이콘을 배치할 수 있다.
- 장식용 아이콘은 `aria-hidden="true"`를 사용한다.
- 의미 전달이 필요한 아이콘은 텍스트 라벨과 함께 사용한다.

CSS 변수 후보:

```css
:root {
  --icon-size: 28px;
  --icon-stroke: 1.8;
  --icon-badge: 46px;
  --radius-card: 18px;
  --radius-section: 28px;
  --shadow-soft: 0 18px 48px rgba(23, 33, 59, .12);
  --shadow-lift: 0 24px 60px rgba(23, 33, 59, .16);
}
```

### 주요 페이지 서브 배너

각 주요 페이지 상단에 같은 디자인 언어를 공유하는 서브 배너를 둔다. 배너는 사진 없이 gradient, icon badge, 짧은 설명, subtle pattern으로 구성한다.

대상 페이지:

- `about.html`: 기도의샘물 소개 배너
- `prayers.html`: 오늘의 기도문 배너
- `night-prayer.html`: 잠잘 때 듣는 밤기도 배너
- `morning-prayer.html`: 아침기도 배너
- `meditation.html`: 말씀 묵상 배너
- `prayer-cards.html`: 말씀·기도 카드 이미지팩 배너
- `prayer-challenge.html`: 30일 기도 챌린지 배너
- `premium-pdf.html`: 프리미엄 기도문 PDF 배너

배너 구성:

- 페이지별 포인트 아이콘
- 짧은 eyebrow
- 명확한 H1
- 1~2문장 설명
- 필요 시 관련 CTA 1개
- 향후 대표 이미지 삽입 위치 주석

공통 클래스 후보:

- `sub-hero`
- `sub-hero-icon`
- `sub-hero-copy`
- `sub-hero-actions`
- `sub-hero--night`
- `sub-hero--morning`
- `sub-hero--premium`

### 도형과 섹션 구분

섹션 구분은 단순 직선보다 부드러운 wave divider, curved shape, subtle band를 사용한다.

- 배경은 아이보리와 연한 베이지를 교차한다.
- 콘텐츠 밀도가 높은 섹션은 화이트 카드 위주로 정리한다.
- 중요한 CTA 섹션은 딥 네이비 배경과 골드 포인트 라인으로 강조한다.
- 말씀 인용 박스는 왼쪽 골드 라인, 부드러운 그림자, 차분한 serif 타이포그래피를 사용한다.
- wave divider는 CSS pseudo-element 또는 inline SVG로 처리하고, 외부 이미지 파일을 만들지 않는다.

공통 클래스 후보:

- `section-band`
- `section-band--ivory`
- `section-band--beige`
- `wave-divider`
- `curved-panel`
- `cta-band`
- `scripture-quote`

### 카드 디자인

콘텐츠 카드는 단순 박스처럼 보이지 않게 한다. 모든 카드가 같은 디자인 시스템 안에 있다고 느껴져야 한다.

공통 카드 규칙:

- 둥근 모서리
- 얇은 베이지/골드 테두리
- 부드러운 그림자
- 상단 작은 카테고리 라벨
- 우측 상단 작은 SVG 아이콘
- hover 시 살짝 떠오르는 효과
- 모바일 1열 정렬

통일 대상:

- `content-card`
- `video-card`
- `journey-card`
- `pdf-product-card`
- `day-card`
- `share-card`
- `hub-card`

카드 hover는 `transform: translateY(-3px)` 정도로 제한한다. 과한 확대, 회전, 튀는 transition은 사용하지 않는다.

### `prayer-challenge.html` 특화 계획

`prayer-challenge.html`은 단순 목록이 아니라 차분한 신앙 여정 프로그램처럼 보여야 한다.

구성:

- 챌린지 유형 카드

주의:

```html
<!-- 향후 30일 기도 챌린지 대표 이미지 삽입 위치 -->
```

### 코드 품질 기준

- `:root`에 색상, 그림자, radius, icon size, gradient token을 추가한다.
- 반복되는 배너, 카드, 아이콘 스타일은 재사용 가능한 class로 정리한다.
- 모바일 최적화를 기본으로 한다.
- decorative SVG와 pseudo-element는 접근성 트리에 노출하지 않는다.
- 텍스트 대비를 충분히 확보한다.
- 버튼 터치 영역은 최소 44px 이상을 유지한다.
- 이미지가 없어도 완성도 있게 보이도록 배경, 여백, 타이포그래피, 카드 구조를 먼저 완성한다.

주요 수정 파일:

- `assets/css/style.css`: 디자인 토큰, hero/banner/card/icon/wave/CTA 시스템 추가
- 주요 HTML 페이지: 서브 배너, 카드 preview, 향후 이미지 삽입 주석 추가
- `assets/js/script.js`: 아이콘 렌더링이 데이터 기반으로 필요할 때만 최소 수정

금지:

- 외부 이미지 URL 추가
- 무단 복제 이미지 사용
- 라이선스 확인 없는 무료 이미지 사용
- `generate_site.py` 실행
- 사용자 화면에 `준비 중`, `샘플`, `임시 이미지` 표시

### 검증 기준

- 이미지 파일 없이도 홈과 주요 페이지가 완성도 있게 보인다.
- 데스크톱/모바일에서 히어로 배너, 서브 배너, 카드, CTA가 겹치지 않는다.
- `prayer-cards.html`의 CSS 기반 카드 preview가 실제 상품 이미지팩처럼 보인다.
- `prayer-challenge.html`의 30일 여정이 차분한 신앙 프로그램처럼 보인다.
- 외부 이미지 URL, 무단 복제 자산, 준비 중/임시 이미지 문구가 사용자 화면에 없다.
- `git diff`에서 의도치 않은 대량 HTML 재생성이나 한글 깨짐이 없다.

## 10. 구현 순서

### Phase 1: 안전 정리

1. `generate_site.py`를 실행하지 않는다는 전제를 유지한다.
2. 현재 한글 정상 HTML 파일만 직접 수정한다.

### Phase 3: 홈 리모델링

1. 히어로 문구를 `기도의샘물` 브랜드 중심으로 정리한다.
2. 오늘의 기도/말씀/영상 섹션을 추가한다.
3. PDF 대표 상품 CTA를 추가한다.
4. 기존 콘텐츠 허브는 상황별 탐색 카드로 재정리한다.

### Phase 4: 기도문/영상 연결

3. `night-prayer.html`에 밤기도 PDF 전환 섹션을 추가한다.

### Phase 5: 디자인 고도화

4. 색상과 여백을 정리해 전체 톤을 세련되게 만든다.

2. 모든 주요 페이지 브라우저 확인
3. 모바일 폭 확인

5. 유튜브 링크 확인
6. 구매/문의 버튼 링크 확인
7. `sitemap.xml` 확인
8. `git diff` 확인

## 11. 수용 기준

리모델링 완료 기준:

- 홈에서 3초 안에 사이트 목적과 주요 행동이 보인다.
- 사용자가 오늘의 기도문을 바로 읽을 수 있다.
- 유튜브 영상으로 이동하는 동선이 명확하다.
- PDF 상품 페이지가 실제 판매 전 단계로 충분히 설득력 있게 보인다.
- 각 PDF 상품 카드에 대상, 구성, 가격대, 샘플, 구매 버튼이 있다.
- 모바일에서 버튼과 카드가 겹치지 않는다.
- 40~70대 사용자가 읽기 어렵지 않은 글자 크기와 대비를 가진다.
- 외부 이미지 저작권 리스크가 없다.
- 사이트 내 모든 주요 CTA가 죽은 링크가 아니다.
