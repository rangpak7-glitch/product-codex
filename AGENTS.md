# AGENTS.md

## 프로젝트 개요

이 프로젝트는 `기도의샘물` 웹사이트입니다.

기도문, 말씀 묵상, 수면기도, 아침기도, 기도 카드, 30일 기도 챌린지, 유튜브 묵상 콘텐츠를 연결해 사용자가 기도와 말씀 콘텐츠를 쉽게 찾고 이어서 묵상할 수 있게 하는 것을 목표로 합니다.

사이트는 현재 정적 HTML/CSS/JavaScript 구조로 운영되지만, 향후 React 등 프론트엔드 프레임워크나 동적 사이트 구조로 확장될 수 있습니다. 따라서 현재 구조를 존중하되, 장기적으로 콘텐츠 재사용성, 컴포넌트화, 데이터 분리, 유튜브 연계 확장을 고려해 작업합니다.

## 핵심 방향

- 브랜드명은 항상 `기도의샘물`로 표기한다.
- 사이트의 중심은 웹페이지 단독 콘텐츠가 아니라 유튜브 채널과 연결되는 묵상 허브다.
- 연계 유튜브 채널: `https://youtube.com/channel/UCFrsilNKJ8xcmn0RUrFz6XQ?si=hM09ebhKwa9m_3W0`
- 각 페이지는 사용자가 기도문을 읽고, 관련 영상으로 이동하고, 다시 관련 기도/묵상 콘텐츠로 이어질 수 있게 구성한다.
- 디자인은 가독성, 시인성, 신뢰감, 세련됨을 최우선으로 한다.
- 기독교 콘텐츠 사이트답게 차분하고 경건하지만, 오래된 교회 게시판처럼 보이지 않도록 현대적인 UI를 유지한다.

## 작업 원칙

- 사용자가 명확한 요청을 하면 제안에 그치지 말고 가능한 범위에서 바로 수정까지 진행한다.
- 기존 파일과 사용자 변경사항을 되돌리지 않는다.
- 현재는 `data/*.js`, `assets/js/script.js`, `assets/css/style.css`, 개별 `.html` 중심 구조이지만, 향후 React 전환 가능성을 고려해 콘텐츠와 표현을 최대한 분리한다.
- 새 기능을 만들 때는 나중에 컴포넌트로 옮기기 쉬운 구조를 선호한다.
- 반복되는 UI는 클래스, 데이터 구조, 렌더링 패턴을 통일한다.
- 한국어 문구는 자연스럽고 따뜻하되 과장하지 않는다.
- 의료, 재정, 심리 위기 관련 기도문은 확정적 치유나 보장을 암시하지 않는다.
- AdSense 검토를 고려해 빈약한 페이지, 과도한 광고 유도, 클릭 유도 문구를 피한다.

## 중요한 주의사항

- `generate_site.py`는 현재 한글이 깨져 보일 수 있다. 무심코 실행하면 정상 HTML의 한글 문구를 덮어쓸 위험이 있다.
- 페이지를 대량 재생성해야 할 경우 먼저 `generate_site.py`의 인코딩과 한글 문구를 복구한 뒤 실행한다.
- 루트의 `style.css`, `script.js`보다 `assets/css/style.css`, `assets/js/script.js`가 HTML에서 실제 로드되는 파일이다.
- 한글 파일은 UTF-8로 유지한다.
- 외부 이미지는 저작권을 확인하지 않고 직접 사용하지 않는다.
- Pinterest 이미지는 무단 삽입하지 말고 디자인 방향, 분위기, 구도 참고용으로만 사용한다.
- 실제 삽입 이미지는 직접 제작, 라이선스가 명확한 무료/구매 이미지, 생성 이미지, 또는 사용 허가된 이미지로 대체한다.
- 사용자가 첨부한 이미지는 별도의 문의와 명시적 승인 없이 절대 임의로 교체, 삭제, 재가공, 대체하지 않는다.

## 유튜브 연계 기준

- 모든 핵심 페이지는 가능한 경우 관련 유튜브 영상으로 자연스럽게 이어지는 동선을 가진다.
- 영상 링크는 단순 외부 링크가 아니라 해당 기도문/묵상 주제와 연결된 콘텐츠로 배치한다.
- 영상 카드에는 최소한 제목, 주제, 관련 말씀, 짧은 설명, 보기 버튼을 포함한다.
- 유튜브 채널 구독 유도는 가능하지만 과도한 반복이나 클릭 강요형 문구는 피한다.
- 웹 콘텐츠와 영상 콘텐츠의 제목/주제/키워드는 서로 검색 시너지가 나도록 맞춘다.

## 콘텐츠 작성 패턴

기도문 데이터는 `data/prayers.js`의 기존 배열 구조를 따른다.

```js
["제목","카테고리","성경구절",["태그1","태그2"],"요약","기도문 본문"]
```

작성 기준:

- 제목은 검색 의도를 포함한다.
- 카테고리는 짧고 직관적으로 쓴다. 예: `평안`, `자녀`, `수면`, `감사`, `관계`, `건강`
- 성경구절은 `요한복음 14장 27절`처럼 한국어 표기한다.
- 태그는 사용자가 실제 검색할 단어 위주로 넣는다.
- 요약은 1문장으로 쓴다.
- 본문은 4~7문장 정도로 작성하고 보통 `아멘.`으로 끝낸다.
- 성경 본문을 길게 인용하지 말고, 말씀의 방향을 묵상 문장으로 풀어 쓴다.
- 특정 교단 논쟁, 과도한 번영신학, 타인을 정죄하는 표현은 피한다.

## 디자인 기준

- 가독성과 시인성을 최우선으로 한다.
- 전체 인상은 깔끔하고 세련된 기독교 묵상 플랫폼이어야 한다.
- 참고 방향은 사랑의교회 사이트처럼 정돈된 정보 구조, 명확한 메뉴, 절제된 색상, 신뢰감 있는 교회/말씀 콘텐츠 톤이다.
- 단, 특정 사이트의 UI를 그대로 복제하지 않는다.
- 아이콘 색상과 디자인은 차분한 네이비, 골드, 화이트, 부드러운 보조색을 중심으로 한다.
- 아이콘은 얇고 정돈된 라인 스타일을 우선한다.
- 카드, 버튼, 섹션은 충분한 여백을 두고 모바일에서도 쉽게 읽히게 만든다.
- 본문 글자는 작게 만들지 않는다. 기도문 사이트 특성상 장년층도 읽기 쉬워야 한다.
- 클릭 가능한 요소는 명확히 보이게 한다.
- 장식보다 읽기 흐름과 콘텐츠 탐색을 우선한다.

## 웹페이지 시각 디자인 방향

`기도의샘물`의 시각 디자인은 단순 텍스트 중심 페이지가 아니라 사진 없이도 완성도 있게 보이는 고급 기독교 말씀·기도 콘텐츠 사이트를 목표로 한다. 실제 사진 이미지가 없는 단계에서는 외부 이미지 URL, 타 사이트 이미지, 라이선스가 불명확한 무료 이미지를 가져오지 말고 CSS, SVG, 그라데이션, 추상 도형, 아이콘, 카드, 섹션 배경만으로 시각 시스템을 구성한다.

전체 분위기는 조용함, 따뜻함, 평안함, 신뢰감, 프리미엄을 기준으로 한다. 대형 교회 홈페이지처럼 공식적인 신뢰감을 주되 교회 행정 사이트처럼 딱딱하지 않게 만들고, 잠잘 때 듣는 기도 콘텐츠에 어울리는 부드러운 빛, 물결, 성경책, 종이 질감, 새벽빛, 촛불의 느낌을 추상적으로 표현한다.

### 컬러와 질감

- 기본 팔레트는 딥 네이비, 아이보리, 부드러운 베이지, 은은한 골드, 세이지 그린을 중심으로 잡는다.
- `:root` CSS 변수로 색상, 그림자, radius, font, section spacing을 관리한다.
- 배경은 아이보리와 연한 베이지를 교차해 페이지가 단조롭게 보이지 않게 한다.
- 중요한 CTA 섹션은 딥 네이비 배경에 얇은 골드 포인트 라인, 은은한 빛 번짐, 절제된 버튼을 사용한다.
- 종이 질감은 실제 이미지 대신 `radial-gradient`, `linear-gradient`, 투명한 반복 패턴, 낮은 opacity의 pseudo-element로 표현한다.
- 한 가지 색 계열만 반복하지 말고 네이비, 아이보리, 골드, 세이지 그린이 역할별로 나뉘도록 한다.

### 히어로 배너

- 히어로는 실제 사진 대신 CSS 그라데이션과 radial light 효과로 먼저 완성한다.
- 딥 네이비와 아이보리, 골드가 자연스럽게 섞인 배경을 사용하고, 은은한 빛 번짐과 물결 같은 곡선을 추가한다.
- 필요하면 `::before`, `::after` pseudo-element로 추상적인 원형 빛, 물결 라인, 얇은 십자가 실루엣을 만든다.
- 히어로 텍스트는 명확하고 짧게 유지하며, 장식 요소가 본문과 CTA를 가리지 않게 한다.
- 향후 실제 이미지를 넣을 수 있도록 HTML에는 `<!-- 향후 히어로 배경 이미지 삽입 위치 -->`처럼 주석을 남길 수 있다. 단, 사용자 화면에는 `준비 중`, `샘플`, `임시 이미지` 같은 표현을 보이지 않게 한다.

### 아이콘 시스템

- 외부 이미지 파일 대신 inline SVG 또는 CSS 기반 아이콘을 우선 사용한다.
- 아이콘은 얇고 우아한 라인 스타일로 통일한다.
- 아이콘 색상은 골드, 네이비, 세이지 그린 중심으로 통일하고, 같은 페이지 안에서 stroke width와 모서리 스타일을 맞춘다.
- 기본 아이콘 세트는 다음 의미를 포함한다.
  - 기도
  - 말씀
  - 수면기도
  - 아침기도
  - 영상묵상
  - 말씀카드
  - 30일 챌린지
  - 프리미엄 PDF
  - 기도제목
  - 문의하기
- 아이콘은 장식만을 위해 과하게 반복하지 말고 카테고리, CTA, 카드 성격을 빠르게 이해시키는 용도로 사용한다.

### 섹션 배너

각 주요 페이지 상단에는 같은 디자인 시스템을 공유하는 작은 서브 배너를 둔다. 배너는 섹션별 포인트 아이콘, 짧은 설명, 부드러운 배경 도형을 포함하되 페이지 제목보다 과하게 튀지 않게 한다.

- `about.html`: 기도의샘물 소개 배너
- `prayers.html`: 오늘의 기도문 배너
- `night-prayer.html`: 잠잘 때 듣는 밤기도 배너
- `morning-prayer.html`: 아침기도 배너
- `meditation.html`: 말씀 묵상 배너
- `prayer-cards.html`: 말씀·기도 카드 이미지팩 배너
- `prayer-challenge.html`: 30일 기도 챌린지 배너
- `premium-pdf.html`: 프리미엄 기도문 PDF 배너

### 도형과 섹션 구분

- 섹션 사이는 단순 직선보다 부드러운 wave divider, curved shape, 둥근 배경 면으로 구분한다.
- wave divider는 SVG 또는 CSS clip-path로 구현하되 모바일에서 높이가 과해지지 않게 한다.
- 말씀 인용 박스는 왼쪽 골드 라인, 아이보리 배경, 부드러운 그림자, 충분한 행간으로 고급스럽게 표현한다.
- 장식 도형은 클릭 요소처럼 보이지 않게 하고, 텍스트 대비와 접근성을 해치지 않아야 한다.

### 카드 디자인

콘텐츠 카드는 단순 박스처럼 보이지 않게 하며 다음 요소를 기본으로 고려한다.

- 둥근 모서리
- 얇은 베이지 또는 골드 계열 테두리
- 부드러운 그림자
- 상단 작은 카테고리 라벨
- 우측 상단 작은 SVG 아이콘
- hover 시 살짝 떠오르는 효과
- 모바일 1열 정렬

카드 hover 효과는 `transform`과 `box-shadow`를 절제해서 사용하고, 긴 기도문 카드에서는 움직임보다 읽기 안정성을 우선한다.

### 말씀·기도 카드 이미지팩 페이지

`prayer-cards.html`은 실제 이미지 파일이 없어도 카드 이미지팩 느낌이 나도록 CSS 기반 미리보기 카드를 구성한다.

- 정사각형 카드 비율을 사용한다.
- 아이보리 배경, 골드 라인, 짧은 말씀 문구, 작은 라벨, 부드러운 그림자를 조합한다.
- 여러 디자인 예시가 보이도록 6개 정도의 CSS 기반 preview card를 배치한다.
- 실제 다운로드 파일이 없다면 다운로드 버튼을 넣지 말고 `자료 문의하기`, `소식 받기`처럼 현재 가능한 동선으로 연결한다.
- 향후 실제 이미지를 넣을 수 있도록 `<!-- 향후 말씀 카드 실제 이미지 삽입 위치 -->` 주석을 남길 수 있다.

### 30일 기도 챌린지 페이지

`prayer-challenge.html`은 프로그램 느낌이 나도록 차분한 신앙 여정형 시각 요소를 사용한다.

- 30일 여정 타임라인과 Day 카드를 구성한다.
- 진행 단계 느낌의 라인을 사용하되 헬스 챌린지처럼 강한 경쟁 분위기는 피한다.
- 평안 회복, 자녀 축복, 감사 회복 등 챌린지 유형 카드를 차분하게 배치한다.
- 세부 Day 카드는 묵상 흐름, 기도 주제, 짧은 말씀 방향이 한눈에 보이게 한다.
- 향후 대표 이미지를 넣을 수 있도록 `<!-- 향후 30일 기도 챌린지 대표 이미지 삽입 위치 -->` 주석을 남길 수 있다.

### 코드 품질과 접근성

- 반복되는 배너, 카드, 아이콘, CTA 스타일은 재사용 가능한 class로 정리한다.
- 모바일에서 카드, 배너, CTA, 타임라인이 자연스럽게 1열 또는 좁은 그리드로 전환되게 한다.
- 텍스트와 배경의 명도 대비를 확보하고, 아이콘만으로 의미를 전달하지 않는다.
- inline SVG에는 필요한 경우 `aria-hidden`, `role`, 접근 가능한 텍스트를 적절히 사용한다.
- 이미지가 없는 현재 상태만으로도 세련되어 보여야 하며, 나중에 직접 제작 이미지나 AI 생성 이미지를 추가하면 더 좋아지는 구조로 만든다.

## 동적 UI 방향

- 각 기도 카테고리의 기도 카드들은 단순 목록보다 카드가 포개진 형태 또는 좌우 슬라이드형 카드 UI를 우선 고려한다.
- 카드 묶음에는 좌/우 이동 아이콘을 제공한다.
- 모바일에서는 스와이프 가능한 구조를 고려한다.
- 현재 정적 JS로 구현하더라도 향후 React 컴포넌트로 옮기기 쉬운 구조로 만든다.
- 카드 UI는 다음 요소를 포함하는 것을 기본으로 한다.
  - 카테고리
  - 제목
  - 짧은 요약
  - 관련 말씀
  - 읽기/영상 보기 버튼
- 애니메이션은 부드럽고 절제되게 사용한다. 과한 움직임으로 기도문 읽기를 방해하지 않는다.

## 이미지와 아이콘 사용 기준

- Pinterest는 이미지, 카드 구도, 색 조합, 분위기 참고용으로만 사용한다.
- Pinterest 이미지를 그대로 다운로드해 사이트에 삽입하지 않는다.
- 실제 삽입 가능한 이미지는 다음 중 하나여야 한다.
  - 직접 제작한 이미지
  - 라이선스가 명확한 무료 이미지
  - 구매 또는 사용 허가를 받은 이미지
  - AI로 생성한 이미지
  - 저작권 문제가 없는 자체 아이콘/일러스트
- 아이콘은 가능한 한 통일된 스타일로 유지한다.
- 아이콘과 이미지는 콘텐츠를 보조해야 하며, 본문 가독성을 방해하지 않아야 한다.
- 사용자가 첨부한 이미지는 원본 의도와 사용 위치를 존중하며, 별도의 문의와 명시적 승인 없이 절대 임의로 바꾸지 않는다.

## SEO / AdSense 기준

- 새 HTML 페이지에는 다음을 포함한다.
  - `<meta charset="UTF-8">`
  - viewport
  - description
  - canonical
  - OG title/description/url/image
  - title
  - Google AdSense account meta
  - AdSense script
  - `assets/css/style.css`
- 새 페이지를 추가하면 `sitemap.xml`에도 URL을 추가한다.
- 정책성 페이지와 문의 페이지 연결을 유지한다.
- 광고 위치는 콘텐츠 읽기를 방해하지 않게 배치한다.
- 광고 클릭을 직접 유도하는 문구는 사용하지 않는다.
- 유튜브와 웹페이지가 같은 주제 키워드로 연결되도록 제목, 설명, 태그를 정리한다.

## 기능 수정 기준

- 현재 검색/렌더링 기능은 `assets/js/script.js`에서 관리한다.
- 콘텐츠 확장은 가능하면 `data/*.js`에 데이터를 추가하고 기존 렌더러를 재사용한다.
- 향후 React 전환을 고려해 데이터 구조를 무리하게 HTML에 박아 넣지 않는다.
- `PRAYERS`, `MEDITATIONS`, `VIDEOS`, `PRAYER_CARDS`, `PRAYER_CHALLENGE` 전역 상수 이름은 현재 구조에서는 유지한다.
- 새 데이터 필드를 추가할 때는 기존 렌더러가 깨지지 않도록 기본값을 고려한다.
- 카드 슬라이더, 카테고리 필터, 관련 영상 추천 등은 재사용 가능한 함수 단위로 만든다.

## 자주 하는 요청 처리 방식

### 기도문 추가

- `data/prayers.js`에 기존 형식으로 추가한다.
- 검색 태그를 충분히 넣는다.
- 관련 영상이 있으면 `data/videos.js`와 연결될 수 있게 주제를 맞춘다.

### 묵상 추가

- `data/meditations.js`에 추가한다.
- 말씀 설명은 짧고 적용 중심으로 작성한다.
- 가능하면 관련 기도문이나 유튜브 영상으로 이어지게 한다.

### 유튜브 영상 추가

- `data/videos.js`에 제목, 테마, 관련 말씀, URL, 설명을 추가한다.
- 웹페이지의 관련 주제와 연결되도록 제목/키워드를 맞춘다.

### 새 페이지 추가

- 기존 HTML 구조를 참고하되, 향후 컴포넌트화 가능성을 고려해 섹션을 명확히 나눈다.
- `sitemap.xml`에 URL을 추가한다.
- 내비게이션 또는 관련 링크에도 반영한다.
- 유튜브 채널 또는 관련 영상으로 이어지는 동선을 고려한다.

### 디자인 개선

- `assets/css/style.css`에서 기존 디자인 체계를 정리하며 개선한다.
- 가독성, 시인성, 모바일 사용성을 우선한다.
- 실제 사진 파일이 없을 때는 CSS/SVG 기반 히어로, 섹션 배너, 카드, 추상 도형, 라인 아이콘만으로 완성도 있는 시각 시스템을 만든다.
- 기도 카드 UI는 포개진 카드, 좌우 이동 버튼, 스와이프 가능한 구조를 우선 고려한다.
- Pinterest와 사랑의교회 사이트는 분위기와 방향 참고용으로만 사용하고 복제하지 않는다.

## 검증 절차

수정 후 최소한 다음을 확인한다.

1. 변경 파일의 한글이 깨지지 않는지 확인한다.
2. 메인 페이지와 수정한 페이지를 브라우저에서 확인한다.
3. 모바일 폭에서 헤더 메뉴, 카드 그리드, 슬라이더, 버튼 줄바꿈이 깨지지 않는지 확인한다.
4. `prayers.html` 검색이 정상 동작하는지 확인한다.
5. 유튜브 링크가 올바른 채널/영상으로 연결되는지 확인한다.
6. 새 페이지나 URL 변경이 있으면 `sitemap.xml`을 확인한다.
7. `git diff`로 의도하지 않은 대량 변경이 없는지 확인한다.
